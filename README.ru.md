# BelBucks

[English](./README.md) | **Русский**

BelBucks — браузерное расширение (Chrome / Edge, Manifest V3), которое автоматически находит цены в BYN на страницах и показывает удобный валютный эквивалент рядом с ними.

Проект ориентирован на белорусские маркетплейсы, каталоги и доски объявлений, с упором на:
- заметный и аккуратный UI;
- быструю работу на динамических страницах;
- гибкие пользовательские настройки;
- живое применение изменений без перезагрузки вкладки.

## Основные возможности

### Конвертация цен
- Автопоиск цен на странице (селекторы + fallback по текстовым узлам).
- Поддержка форматов вроде `1 500,00 BYN`, `1.500 р.`, `от 1105,60 ƃ`, `15.5 коп.`.
- Показ в формате бейджа: `≈ $100.00` и опционально в 2 валютах: `≈ $100.00 · €92.00`.
- Режимы отображения: `inline` и `tooltip`.

### Источники курса
- `NBRB` (официальный курс).
- `BankAverage` (средний по банкам, агрегированный фид в MVP).
- `BankBest` (лучший по банкам, агрегированный фид в MVP).
- `BankSpecific` (MVP-режим через доступный банковский фид).
- `Custom` (ручной курс).

### Кастомизация
- Основная и вторая валюта (`USD`, `EUR`, `PLN`, `RUB`).
- Наценка банка (`-5% ... +10%`).
- Цвет карточки и цвет текста.
- Режим `Auto theme` (подбор цвета по сайту: например, `av.by`/`onliner.by`).
- Белый/черный список доменов.

### Трекер цены
- Сохраняет историю цен по страницам.
- Показывает мини-график (sparkline) в popup.
- Использует пакетную запись с дебаунсом, чтобы не упираться в лимиты `chrome.storage`.

## Технологии

- `TypeScript`
- `React` (UI popup/options)
- `Vite` (сборка)
- `Manifest V3`
- `chrome.storage.sync` (пользовательские настройки)
- `chrome.storage.local` (кэш курсов, история цен)
- `MutationObserver` (реакция на динамическую подгрузку)

## Структура проекта

- `src/content/contentScript.ts` — поиск и рендер цен на сайтах.
- `src/content/presets.ts` — пресеты селекторов по площадкам.
- `src/background/serviceWorker.ts` — обновление и кэш курсов.
- `src/shared/priceParser.ts` — парсер цен в BYN.
- `src/shared/converter.ts` — конвертация и форматирование.
- `src/shared/rates/` — провайдеры курсов.
- `src/shared/storage.ts` — чтение/запись настроек и кэша.
- `src/shared/priceTracker.ts` — логика трекера цены.
- `src/ui/options/` — полноценные настройки.
- `src/ui/popup/` — быстрые настройки и трекер.

## Установка для разработки

### 1) Установка зависимостей
```bash
npm install
```

### 2) Сборка расширения
```bash
npm run build
```

Готовая сборка будет в `dist/`.

### 3) Подключение в браузере
- Открыть страницу расширений:
  - Chrome: `chrome://extensions`
  - Edge: `edge://extensions`
- Включить `Developer mode`.
- Нажать `Load unpacked`.
- Выбрать папку `dist/`.

## Команды

- `npm run build` — production build.
- `npm test` — запуск unit-тестов (`vitest`).

## API курсов (MVP)

- НБРБ:  
  `https://www.nbrb.by/api/exrates/rates/{CUR}?parammode=2`

- Банковский фид (используется в MVP-режимах BankAverage/BankBest):  
  `https://belarusbank.by/api/kursExchange?city=...`

> Примечание: в текущем MVP режимы “по банкам” реализованы через доступный агрегированный банковский фид. Позже можно подключить отдельные адаптеры для нескольких источников (включая Myfin) в рамках того же слоя провайдеров.

## Поведение в реальном времени

- Изменения настроек применяются на текущей вкладке без перезагрузки:
  - перерисовка бейджей;
  - live-обновление цветов;
  - авто-refresh курсов при смене провайдера/валют.
- Запись настроек и истории защищена от quota-ошибок (дебаунс + пакетирование).

## Известные ограничения MVP

- На некоторых сайтах с очень нестабильным DOM могут требоваться дополнительные точечные селекторы.
- Price Tracker в MVP сохраняет “репрезентативную” цену страницы (а не отдельный трек по каждому товару в листинге).
- Часть банковских режимов пока опирается на общий фид, а не на полноценный мульти-агрегатор.

## Troubleshooting

### Не видно бейджей цен
- Проверьте, что расширение включено.
- Убедитесь, что домен не в blacklist.
- Нажмите `Reload` в `chrome://extensions`.
- Обновите страницу (`Ctrl+F5`).

### Изменения цветов/настроек не применяются
- Проверьте, не включен ли `Auto theme` (в этом режиме ручные цвета отключены).
- Нажмите в popup `Обновить курс` при смене провайдера.

### Ошибки quota в extensions
- В проекте уже применены дебаунс/пакетирование записей.
- Если старые ошибки остались в UI браузера, очистите список ошибок и перезагрузите расширение.
